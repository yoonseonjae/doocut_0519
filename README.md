# 인생DOO컷 (doocut_0519)

인생DOO컷 데모 시스템의 실행 및 설정 가이드입니다.

## 1. 환경 변수 설정 (.env 파일 위치)

음성 인식을 위한 OpenAI API 키 설정이 필요합니다. 보안상 깃허브에는 업로드되지 않도록 설정되어 있습니다.
아래 지정된 위치에 `.env` 파일을 직접 생성하고 키를 입력해 주세요.

- **파일 생성 위치:** `src/voice_processing/resource/.env`
- **파일 내용 예시:**
  ```env
  OPENAI_API_KEY="sk-여러분의_openai_api_키를_입력하세요"
  ```

## 2. 도커(Docker) 환경 접속 방법

도커 환경에서 ROS2 패키지를 빌드하고 실행하려면, 터미널을 열고 도커 컨테이너 내부로 접속해야 합니다.

```bash
# 실행 중인 도커 컨테이너 내부로 접속 (bash 쉘 열기)
# (주의: 사용 중이신 컨테이너 이름에 맞게 '<도커_컨테이너_이름>' 부분을 수정해주세요)
docker exec -it <도커_컨테이너_이름> bash

# 만약 아직 도커 컨테이너가 실행되지 않았다면 docker run 명령어로 컨테이너를 먼저 실행해야 합니다.
```

## 3. 프로그램 빌드 및 실행 방법 (터미널 명령어)

프로그램을 실행하기 위해서는 도커 컨테이너 터미널 내에서 아래 명령어들을 순차적으로 실행합니다.

### 터미널 1: 패키지 빌드 및 전체 시스템 실행

로봇 제어, 객체 인식(비전), 음성 처리, 사진 매니저 노드를 모두 한 번에 켭니다.

```bash
# 1. 워크스페이스 디렉토리로 이동
cd ~/doocut_ws

# 2. 패키지 빌드 (코드를 수정했을 경우 필수)
colcon build --symlink-install

# 3. 환경 변수 적용 (새로운 터미널을 열 때마다 실행해야 함)
source install/setup.bash

# 4. 전체 시스템 런치 실행 (로봇 포함)
ros2 launch doocut_bringup system.launch.py
```

### 터미널 2 (선택): 비전 및 음성 기능만 단독 테스트 시

로봇 연결 없이 카메라와 마이크(음성) 노드만 테스트하고 싶을 때 새로운 터미널을 열어 사용합니다.

```bash
# 도커 접속 후 워크스페이스 이동 및 환경 변수 적용
cd ~/doocut_ws
source install/setup.bash

# 비전 및 음성 테스트용 런치 실행
ros2 launch doocut_bringup vision_only.launch.py
```
